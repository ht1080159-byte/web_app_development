# 產品需求文件 (PRD) - 英文口說練習系統

## 1. 專案概述
### 背景與動機
現代人學習英文時，往往偏重讀與寫，而在口說部分缺乏練習對象與即時回饋。為了幫助使用者突破開口說英文的障礙，本專案旨在開發一套英文口說練習系統，提供標準發音範例與自動化的發音判讀功能，讓學習者能隨時隨地練習並獲得客觀的評分與建議。

### 目標用戶
- 想提升日常英文口語表達能力的學生與上班族。
- 準備多益 (TOEIC) 口說、托福 (TOEFL)、雅思 (IELTS) 等英文檢定考試的考生。
- 缺乏自信在真人面前開口說英文的初學者。

### 核心價值主張
提供一個低壓力、隨時可用的口說練習環境，結合語音辨識技術，給予即時、精準的發音回饋，有效提升使用者的口說能力與自信。

## 2. 功能需求

### 主要功能清單
1. **聆聽範例**：播放單字或句子的標準英文發音。
2. **錄音與判讀是否正確**：錄製使用者的發音，並進行語音辨識與評分。
3. **課程與主題選擇**：依難度、情境分類提供不同的口說練習題材。
4. **學習紀錄與追蹤**：記錄使用者的練習歷史、分數與進度。
5. **發音弱點分析**：針對發音不標準的單字或音節給予具體標示。

### 使用者故事 (User Stories)
- **作為** 一名學習者，**我希望** 能夠聆聽標準的英文發音範例，**以便** 知道正確的唸法。
- **作為** 一名學習者，**我希望** 能夠錄下自己的發音並讓系統判讀是否正確，**以便** 了解自己的發音準確度。
- **作為** 一名學習者，**我希望** 能夠根據不同情境（如職場、旅遊）選擇練習題目，**以便** 學習實用的口語表達。
- **作為** 一名學習者，**我希望** 能夠查看過去的練習紀錄與成績走勢，**以便** 追蹤自己的學習進步狀況。
- **作為** 一名學習者，**我希望** 系統能明確指出我念錯的單字或發音弱點，**以便** 進行針對性的改善。

## 3. 非功能需求
### 技術限制
- 前端框架：Jinja2 模板搭配 HTML/CSS/JavaScript (支援瀏覽器錄音 API)。
- 後端框架：Python Flask。
- 資料庫：SQLite (用於儲存使用者資料、題目、練習紀錄)。
- 語音技術：串接第三方 API (如 Google Web Speech API, OpenAI Whisper, 或其他 TTS/STT 服務)。

### 效能與安全考量
- **效能**：語音判讀的回饋時間應在 2~3 秒內完成，避免使用者等待過久。
- **安全**：保護使用者的語音資料，錄音檔原則上分析完畢後不應長久保存，或僅限使用者本人存取。密碼與個人資料應妥善加密儲存。

## 4. MVP 範圍 (Minimum Viable Product)

| 功能層級 | 功能描述 |
| :--- | :--- |
| **Must Have (必須有)** | 1. 播放預設的英文句子聲音檔 (聆聽)<br>2. 網頁端錄音功能<br>3. 串接 API 判讀錄音內容與標準答案的相符程度，並給出基本分數<br>4. 提供單一基礎題庫列表供選擇 |
| **Should Have (應該有)** | 1. 使用者註冊與登入系統<br>2. 儲存個人專屬的練習紀錄與分數<br>3. 顯示發音錯誤或遺漏單字的具體回饋 |
| **Nice to Have (可以有)** | 1. 語音弱點分析與圖表化的學習進度面板<br>2. 支援客製化或匯入專屬題目<br>3. 全球排行榜或社群分享機制 |
